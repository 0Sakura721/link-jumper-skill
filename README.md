## 设计理念

`link-jumper` 是一套 **通用方法论型的思维框架**。它的核心设计原则是：

> **不预设任何平台，不锁死任何配置，AI 依据上下文自主判断如何处理链接。**

与其他领域技能中常见的「绑定特定平台/组件」的做法不同，`link-jumper` 的定位是一套 **思维方法**——通过定义清晰的推理链路和行为模式，让 AI 拥有自主判断「当前环境是什么、该用什么工具、该如何构造目标」的能力。

### 为何选择这种设计？

| 传统做法 | link-jumper 的做法 |
|----------|-------------------|
| 预设 `http://` → 打开浏览器 | **AI 分析链接类型 + 用户意图 → 决策最佳打开方式** |
| 绑定特定平台的 API（如 URL Scheme） | **抽象为「协议 → 动作」映射，由 AI 自行推断环境适配** |
| 写死搜索平台（如仅使用特定引擎） | **AI 根据用户言论和历史推测目标平台** |
| 视频播放固定使用某播放器 | **AI 探测环境中的可用播放器，动态选择** |

## 能力概览

| 场景 | 行为 |
|------|------|
| **🔗 链接跳转** | AI 使用系统 Intent 路由或终端命令（如 `xdg-open`）打开目标链接 |
| **🔍 搜索跳转** | AI 分析用户意图 → 推理目标平台 → 构造搜索 URL 并执行打开 |
| **🎬 视频播放** | AI 提取流地址 → 使用本地播放器（如 `mpv`）+ 防盗链 Referer 头处理 |
| **📧 特殊协议** | `mailto:`、`tel:` 等直接路由到系统对应 App |

## 工作原理

### 三种行为模式（定义于 SKILL.md）

```
行为模式 A: 链接跳转
  用户请求 → AI 提取链接 → 识别协议类型（http/https/mailto/tel/其他）
  → 调用系统 Intent 或命令行打开 → 完成

行为模式 B: 搜索跳转
  用户请求关键词 → AI 推理目标平台 → 构造搜索引擎 URL
  → 调用系统 Intent 或命令行打开 → 完成

行为模式 C: 视频播放
  用户请求视频 → AI 提取流地址 → 检查本地播放器可用性
  → 添加防盗链头 → 调用 mpv / 其他播放器 → 完成
```

### 四条思维原则

1. **环境探测优先** — AI 在处理链接前，先探测当前环境（Android/Linux），再选择对应工具
2. **用户意图推导** — 对于模糊请求（如「打开那个视频」），AI 主动在对话历史中查找最佳匹配
3. **错误恢复** — 若首选方式失败（如播放器未安装），AI 应尝试备选方案并告知用户
4. **不越界** — 对于 `mailto:`、`tel:` 等特殊协议，只路由到系统对应 App，不尝试解析或修改内容

## 环境支持

| 环境 | 实现方式 | 依赖 |
|------|---------|------|
| **Android** | 系统 Intent 路由（如 `ACTION_VIEW`） | 需具备 Intent 发送能力 |
| **Linux** | 终端命令执行（如 `xdg-open`、`mpv`） | 需具备终端命令执行能力；视频播放可选需 `mpv` |
| **其他环境** | AI 自动推理可用工具 | 本 Skill 不绑定特定平台，AI 可扩展支持 |

## 使用前提

### 必需能力

| 环境 | 所需能力 | 说明 |
|------|---------|------|
| **Android** | 系统 Intent 发送能力 | 用于发送 `ACTION_VIEW` 等 Intent 跳转链接 |
| **Linux** | 终端命令执行能力 | 用于执行 `xdg-open`、`mpv` 等命令 |

### 可选依赖

- **`mpv`**（Linux）— 用于视频播放场景。安装：`sudo apt install mpv`
- **`xdg-utils`**（Linux）— 通常预装，用于 `xdg-open` 打开链接

## 限制与坑点

> 以下内容参考社区经验总结，使用前建议阅读。

### ⚠️ 已知限制

| # | 限制 | 说明 | 缓解策略 |
|---|------|------|---------|
| 1 | **平台探测有时延** | 每次执行时 AI 需要先探测环境，增加少量推理延迟 | 可预先告知 AI 当前环境 |
| 2 | **视频播放依赖 mpv** | Linux 环境下若不安装 mpv，视频场景无法执行 | 安装 `mpv` 或使用 `ffmpeg` 处理 |
| 3 | **搜索平台推测不 100% 准确** | AI 推测用户意图时的目标平台可能有偏差 | 用户可明确指定平台 |
| 4 | **短链接重定向处理** | 部分短链接需要多次重定向才能到达目标 | AI 需要主动跟踪重定向 |

### 🔧 常见问题

- **Q: 为什么我让 AI 打开链接，但它搜索了？**
  A: 说明 AI 将用户请求分类到了「行为模式 B（搜索跳转）」而非「A（链接跳转）」。可明确告诉 AI「直接打开这个链接」。

- **Q: 播放视频时说 mpv 未安装？**
  A: 请先在 Linux 环境中安装 mpv：`sudo apt update && sudo apt install mpv`。

- **Q: 在 Android 上 mailto 链接没反应？**
  A: 检查系统是否有默认邮件客户端，或使用 Intent 手动发送 `ACTION_SENDTO`。
