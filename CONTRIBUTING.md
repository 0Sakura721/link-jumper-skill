## <span id="english">🌐 English</span>

### How to Contribute

1. **Fork the repo** and create your branch from `main`
2. **Make your changes** — improve the methodology, add test cases, fix bugs
3. **Test** — run through the relevant test cases in `test_cases.md`
4. **Submit a Pull Request** with a clear description of what you changed and why

### What We're Looking For

- **New platform integration guides** — how AI should handle links on a specific platform
- **Better fallback strategies** — more graceful degradation patterns
- **Edge case handling** — scenarios we haven't thought of
- **Documentation improvements** — clearer explanations, better examples

### Style Guide

- Keep the methodology **platform-agnostic** — don't hardcode platform specifics
- Use **clear, actionable language** — AI needs to understand exactly what to do
- Maintain **bilingual consistency** — update both English and Chinese docs

## <span id="chinese">🇨🇳 中文</span>

### 如何贡献

1. **Fork 仓库**并从 `main` 分支创建你的新分支
2. **修改代码** — 改进方法论、添加测试用例、修复问题
3. **测试** — 运行 `test_cases.md` 中的相关测试
4. **提交 PR** — 清晰描述你修改了什么以及为什么

### 我们在找什么

- **新平台集成指南** — AI 应如何处理特定平台上的链接
- **更好的降级策略** — 更优雅的失败降级模式
- **边界情况处理** — 我们还没想到的场景
- **文档改进** — 更清晰的解释、更好的示例

### 风格指南

- 保持方法论**平台无关** — 不要硬编码平台特定内容
- 使用**清晰可操作的语言** — AI 需要确切知道该做什么
- 保持**双语一致性** — 同时更新中英文文档
